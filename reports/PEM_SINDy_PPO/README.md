# PEM + SINDy-PI + PPO Offline Identification Benchmark

This folder stores artifacts generated by:

```bash
python host/offline_id_pem_sindy_ppo.py \
  --csv host/run_logs/chrono_run_1.finalized.csv \
  --meta host/run_logs/chrono_run_1.meta.json \
  --outdir reports/PEM_SINDy_PPO
```

or from the stack menu:

```bash
bash host/pendulum_stack.sh
# menu -> 8) Offline Benchmark (PEM+SINDy-PI+PPO)
```

## Stage-wise roles

1. **Stage 1 (PEM nominal model)**
   - Uses the physically constrained pendulum ODE and identifies nominal parameters with nonlinear least squares.
   - Primary integration path: **native irregular timestamps with RK45**.
   - Baseline comparison path: **uniform resampling to `dt_mean` + RK4**.

2. **Stage 2 (Residual discovery with SINDy-PI)**
   - Learns only residual acceleration dynamics:
     `r(t) = omega_dot_measured - omega_dot_nominal`.
   - Candidate terms include nonlinear friction/gravity/input interactions.
   - Final model is additive (`nominal + residual`) and preserves nominal interpretability.

3. **Stage 3 (PPO parameter proposal)**
   - PPO is used as an **offline parameter proposal optimizer**, not as a controller.
   - Each episode proposes one parameter vector, then receives rollout-level reward based on trajectory mismatch + penalties.

## Notes on reproducibility

- Use only IMU-derived channels from finalized logs:
  `theta_imu_filtered_unwrapped`, `omega_imu_filtered`, and optional `alpha_from_linear_accel_filtered`.
- Keep train/validation split chronological to avoid leakage.
- Keep `config_used.json` with each run to track exact settings.
